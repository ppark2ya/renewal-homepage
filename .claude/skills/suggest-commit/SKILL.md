# 커밋 메시지 추천

현재 git 변경사항을 분석하여 커밋 메시지를 추천해줘.

## 분석 순서

1. `git status`로 변경된 파일 확인
2. `git diff`로 변경 내용 확인
3. `git log --oneline -10`으로 기존 커밋 스타일 참고

## 커밋 규칙

프로젝트 컨벤션을 따를 것:
- `feat:` 새 기능
- `fix:` 버그 수정
- `refactor:` 리팩토링
- `style:` 스타일/포맷팅
- `docs:` 문서
- `chore:` 유지보수

## 출력 형식

- **추천 1 (메인)**: 가장 적합한 커밋 메시지
- **추천 2-3**: 대안 옵션
- 변경 범위가 크면 분리 커밋 방안도 제안
- 한국어로 커밋 메시지 작성
